# AI Research Assistant - 智能研究助手

## 📌 项目简介
一个基于 **ReAct Agent** 的研究助手，能够自动规划研究任务、搜索学术论文、分析文献、生成代码，并输出结构化研究报告。系统内置评估模块，可量化研究质量和执行效率，支持持续迭代优化。

## 🚀 核心技术
- **任务规划**：LLM将复杂问题分解为原子化子任务（3-8步）
- **ReAct Agent**：LangGraph实现的多轮工具调用框架
- **工具集成**：arXiv搜索、GitHub搜索、论文分析、代码生成
- **记忆机制**：滑动窗口+摘要的短期记忆，支持上下文感知
- **流式执行**：生成器模式实现执行过程实时反馈
- **质量评估**：基于规则+本地LLM的报告质量打分，数据完整性检查，过程效能分析
- **日志追踪**：结构化日志记录，支持控制台输出与文件持久化

## 🛠️ 技术栈
- **语言模型**：通义千问 qwen3-max + 本地 Ollama (qwen2.5:7b)
- **Agent框架**：LangGraph + ReAct模式
- **前端界面**：Streamlit（支持历史记录、流式展示）
- **工具封装**：arXiv API、PyGithub、自定义分析工具
- **日志与评估**：Python logging、JSON指标记录

## 💡 核心亮点
1. **自主规划**：Planner将模糊问题分解为可执行的原子任务，确保执行方向明确
2. **短期记忆**：滑动窗口+摘要，让Agent记住研究进展，避免重复劳动
3. **流式输出**：执行过程实时展示，用户体验友好
4. **多工具协同**：支持论文搜索、代码生成、报告撰写全流程
5. **历史记录**：自动保存每次研究，支持回顾和报告下载
6. **质量评估**：对研究报告进行多维度评分，量化完整性、逻辑性和格式，并提供改进建议
7. **可观测性**：结构化日志记录执行过程、工具调用、错误重试，支持事后分析与优化

## 🎯 应用场景
- 学术文献调研
- 技术方案预研
- 论文阅读辅助
- 代码示例生成
- Agent系统性能评估与优化

## 📊 评估示例
系统会为每次研究生成综合评分（0-10分），包括：
- 报告质量（章节覆盖、代码块、参考文献等）
- 数据完整性（论文/代码/GitHub/对比分析数量）
- 过程效能（工具调用成功率、重试次数、耗时）
- 改进建议（如“增加代码示例”“补充参考文献”）

评估结果会写入日志，为后续迭代提供量化依据。
